Demystifying the Journey of "google.com" in Your Browser: A Deep Dive into Internet Infrastructure

Posted by Abraham Benson on September 15, 2023 | 0 Likes


What happens when you type google.com in your browser and press Enter

Have you ever wondered what happens behind the scenes when you type "google.com" in your browser and hit Enter? The seemingly simple action triggers a complex series of interactions involving various components of the internet infrastructure. In this article, we will take a closer look at each step of the journey, emphasizing the core areas of DNS, TCP/IP, firewalls, HTTPS/SSL, load-balancers, web servers, application servers, and databases. Let's dive in!

DNS (Domain Name System)

As you enter "google.com" in your browser, the first key player in the process is the Domain Name System (DNS). Your computer sends a DNS request to a DNS resolver, asking for the IP address associated with "google.com." The resolver navigates through a hierarchy of DNS servers to obtain the IP address. Here's a simplified breakdown:

  • Your computer queries a local DNS resolver (usually provided by your ISP or configured manually).
  • The resolver checks its cache. If it doesn't have the IP address, it sends a request to the root DNS server.
  • The root server directs the resolver to the appropriate top-level domain (TLD) server responsible for ".com" domains.
  • The TLD server, in turn, points the resolver to the authoritative DNS server responsible for "google.com."
  • The authoritative server provides the IP address back to the resolver, which then caches it for future use.

TCP/IP (Transmission Control Protocol/Internet Protocol)

Once the browser has obtained the IP address of "google.com" from the DNS resolution, it proceeds with establishing a connection using TCP/IP. TCP breaks the data into packets, while IP handles routing and addressing. The browser initiates a TCP connection with the web server at the IP address obtained.

Firewall

In the journey to "google.com," firewalls act as security gatekeepers. They monitor and filter network traffic to protect against unauthorized access and potential threats. When the browser sends a request to the web server, it passes through firewalls that examine the request for any potential security risks before allowing it to proceed.

HTTPS/SSL (Hypertext Transfer Protocol Secure/Secure Sockets Layer)

To ensure secure communication, websites like Google utilize HTTPS/SSL encryption. As the connection is established between your browser and the web server, the server presents a digital certificate verifying its identity. Your browser checks the certificate's authenticity and initiates an SSL handshake to establish an encrypted connection. This encryption protects your data from interception and maintains privacy.

Load Balancer

Google, being a highly trafficked website, employs load-balancers to distribute incoming requests across multiple servers. Load-balancers help optimize performance, ensure high availability, and prevent any single server from being overwhelmed. When you visit "google.com," the load-balancer routes your request to an available web server to handle the incoming traffic.

Web Server

Once the request reaches a web server, it processes the request and generates the appropriate response. The web server retrieves the requested web page and sends it back to your browser over the established connection. Google employs robust web servers capable of handling millions of simultaneous requests efficiently.

Application Server

In more complex web applications, an application server comes into play. It processes dynamic content, executes server-side code, and interacts with databases. For example, Google's search functionality involves an application server that handles the search request, processes the query, and retrieves relevant results.
